Unity3dPlugins载入

Oct 20, 2016 | Hits

http://stackoverflow.com/questions/15643549/why-dllimport-doesnt-work-with-internal

“__Internal” is to be used only when the symbol is defined in the executable. You seem to have a dynamic library instead, and then you must use the library name.

参考unity3d文档。

https://docs.unity3d.com/Manual/NativePlugins.html

在iOS平台和Xbox 360平台

plugins被静态链接到可执行文件中，直接用__Internal即可。

其他平台

plusins的动态库通过名字来区分。

在Mac下载入以.bundle为后缀的文件。
参考如下：
http://answers.unity3d.com/questions/23615/how-to-make-unity-find-dylib-files.html
http://www.voidbred.com/blog/2014/10/calling-c-functions-from-unity-on-mac-os-xios/

在android下载入.so为后缀的文件。

在iOS下载入.a为后缀的文件。

在windows下载入.dll为后缀的文件。
